## 1. DefectShield — Agent de conformité défauts bâtiment

**Score: 18/20** (Faisabilité: 5, Marché: 5, Originalité: 4, Synergie: 4)

**One-liner:** Agent IA qui traque automatiquement les délais légaux de notification de défauts pour les propriétaires suisses.

**Le problème:** Depuis le 1er janvier 2026, la loi suisse impose un délai de 60 jours pour signaler les défauts de construction (art. 367 al. 1bis CO). Avant c'était 7-10 jours. Les propriétaires et régies ne connaissent pas ce changement, risquent de perdre leurs droits.

**La solution:** Un agent qui (1) ingère les rapports de diagnostic Batiscan, (2) détecte automatiquement les défauts couverts par la garantie, (3) calcule les deadlines légales, (4) envoie des alertes avant expiration, (5) génère la lettre de notification conforme.

**Pourquoi Robin:** Batiscan identifie déjà les défauts. SwissBuilding a le moteur de conformité OTConst/CFST. C'est juste une couche de logique temporelle + alertes par-dessus.

**Concurrence:** Aucun outil proptech suisse ne fait ça. Les avocats le font manuellement à CHF 300/h.

**Premier pas:** Ajouter un module `defect_timeline.py` à SwissBuilding qui croise les diagnostics avec les dates d'achat et calcule les deadlines.

## 2. BuildingVoice — Assistant vocal pour techniciens terrain

**Score: 16/20** (Faisabilité: 4, Marché: 4, Originalité: 4, Synergie: 4)

**One-liner:** Un agent vocal qui accompagne les techniciens de diagnostic pendant leurs inspections terrain.

**Le problème:** Les techniciens Batiscan inspectent des bâtiments avec les mains occupées (outils, échantillons, lampe). Ils prennent des notes sur papier ou dictaphone, puis ressaisissent tout au bureau. 45% du temps est du travail administratif post-terrain.

**La solution:** Un agent vocal (OpenClaw + whisper) sur le téléphone du technicien. Il dicte : "Amiante friable trouvée dans le faux plafond salle 203, état dégradé, priorité haute". L'agent crée automatiquement l'observation dans Batiscan avec la bonne classification, le bon code OTConst, et la photo géolocalisée.

**Pourquoi Robin:** Batiscan a déjà le modèle de données, la classification des polluants, les codes de conformité. L'API existe. Il manque juste l'interface vocale.

**Concurrence:** PlanRadar fait du suivi chantier vocal mais PAS de diagnostic polluants. Aucun outil vocal spécialisé polluants en Suisse.

**Premier pas:** Un skill OpenClaw qui écoute via micro, transcrit, et appelle l'API Batiscan pour créer une observation.

## 3. RénoPredict — Simulateur de coûts de désamiantage IA

**Score: 17/20** (Faisabilité: 4, Marché: 5, Originalité: 4, Synergie: 4)

**One-liner:** Prédiction instantanée du coût de désamiantage/décontamination basée sur les diagnostics existants + données historiques.

**Le problème:** Un propriétaire reçoit un diagnostic "amiante dans les joints de fenêtres". Sa première question : "Combien ça va me coûter ?" Aujourd'hui il faut attendre 2-4 semaines pour un devis d'entreprise spécialisée. Pendant ce temps, il ne peut pas budgéter, ni décider, ni négocier un achat.

**La solution:** À partir du diagnostic Batiscan (type de polluant, surface, état, localisation), l'IA prédit une fourchette de coût en croisant avec les prix historiques de Robin (10+ ans de données Batiscan) + les tarifs régionaux. Output : "Estimation : CHF 12'000-18'000, délai 2-3 semaines, 3 entreprises recommandées dans votre canton."

**Pourquoi Robin:** Batiscan a 10+ ans de données de diagnostics réels. C'est la donnée propriétaire que personne d'autre n'a — le moat absolu (cf. recherche : "proprietary data = structural advantage").

**Concurrence:** Aucun outil ne fait de prédiction de coût basée sur des diagnostics polluants réels. Les devis sont manuels.

**Premier pas:** Exporter les données historiques Batiscan (type/surface/coût réel), entraîner un modèle de régression simple, exposer via API SwissBuilding.

## 4. NégoImmo — Coach de négociation immobilière suisse

**Score: 15/20** (Faisabilité: 5, Marché: 3, Originalité: 3, Synergie: 4)

**One-liner:** NegotiateAI spécialisé pour les transactions immobilières suisses.

**Le problème:** En Suisse, 80% des acheteurs ne négocient pas le prix d'achat parce qu'ils ne savent pas comment, ne connaissent pas le marché, et ont peur de perdre le bien. Résultat : ils surpayent de 5-15%.

**La solution:** NegotiateAI avec des presets immobiliers suisses : (1) analyse du prix vs comparables locaux (via registre foncier public), (2) simulation de la négociation avec le vendeur/courtier, (3) playbook personnalisé : quoi dire, quand, comment. Inclut les spécificités suisses (droit de préemption, notaire obligatoire, PPE).

**Pourquoi Robin:** NegotiateAI existe déjà (254 tests, CLI fonctionnel). Il suffit d'ajouter des scénarios immobiliers suisses et de connecter tavily pour les données de marché.

**Concurrence:** Zillow a lancé un AI feature pour les agents US. Rien en Suisse pour les ACHETEURS.

**Premier pas:** 3 presets NegotiateAI : achat_appartement, achat_maison, négociation_rénovation. Connecter tavily pour les prix de marché.

## 5. PortfolioSentinel — Surveillance continue de portefeuille immobilier

**Score: 16/20** (Faisabilité: 4, Marché: 4, Originalité: 5, Synergie: 3)

**One-liner:** Un agent qui surveille en continu la santé réglementaire et financière d'un portefeuille immobilier.

**Le problème:** Un gestionnaire de portefeuille (régie, caisse de pension, fonds) a 50-500 bâtiments. À chaque changement de loi (nouveau délai de 60 jours, nouvelles normes CFST, nouvelle réglementation énergétique), il devrait re-analyser CHAQUE bâtiment. Personne ne le fait — trop coûteux, trop long.

**La solution:** Un agent OpenClaw qui (1) surveille les changements réglementaires suisses via tavily (RSS des publications fédérales/cantonales), (2) croise avec les données du portefeuille dans SwissBuilding, (3) identifie automatiquement quels bâtiments sont impactés, (4) génère un rapport d'impact avec recommandations et deadlines.

**Pourquoi Robin:** SwissBuilding a déjà le moteur de conformité + le module portfolio. L'agent OpenClaw surveille les régulations. C'est la combinaison parfaite des outils installés aujourd'hui.

**Concurrence:** Les cabinets d'avocats font ça manuellement (CHF 500+/h). Wüest Partner fait de la data immobilière mais PAS de veille réglementaire automatisée.

**Premier pas:** Un cron OpenClaw quotidien qui cherche "nouvelles régulations construction Suisse" via tavily, compare avec les règles SwissBuilding, et alerte si changement.

## 6. BiasProof — Certification anti-biais pour décisions immobilières

**Score: 14/20** (Faisabilité: 3, Marché: 3, Originalité: 5, Synergie: 3)

**One-liner:** Un outil qui certifie que les décisions de gestion immobilière sont prises sans biais cognitifs.

**Le problème:** Les caisses de pension et fonds immobiliers doivent justifier leurs décisions d'investissement/désinvestissement auprès des régulateurs (FINMA). "Pourquoi avez-vous vendu cet immeuble et pas l'autre ?" Aujourd'hui : des slides PowerPoint subjectifs. Demain : des décisions traçables et auditées.

**La solution:** Chaque décision (acheter, vendre, rénover, attendre) passe par /audit-decision. L'outil produit un certificat : "Cette décision a été analysée pour les biais suivants : ancrage ✅ non détecté, confirmation ⚠️ détecté et corrigé, sunk cost ✅ non détecté. Score de robustesse : 82/100."

**Pourquoi Robin:** /audit-decision existe déjà comme skill. SwissBuilding a le contexte immobilier. C'est la collision audit épistémique × compliance financière.

**Concurrence:** ClearView fait de la détection de biais pour la finance. Personne ne le fait spécifiquement pour l'immobilier + conformité FINMA.

**Premier pas:** Un module SwissBuilding qui appelle /audit-decision sur chaque décision de portefeuille et génère un PDF "certificat anti-biais".

## 7. ChantierMiroir — Jumeau numérique de chantier de décontamination

**Score: 13/20** (Faisabilité: 2, Marché: 4, Originalité: 5, Synergie: 2)

**One-liner:** Suivi en temps réel d'un chantier de désamiantage avec agents IA qui simulent les risques.

**Le problème:** Un chantier de décontamination amiante est dangereux et très réglementé. Un retard ou une erreur = arrêt de chantier par la SUVA, amendes, risques sanitaires. Le suivi est papier + visites ponctuelles.

**La solution:** Des capteurs IoT (poussière, pression, température) + photos quotidiennes + données Batiscan alimentent un jumeau numérique. Des agents IA (TinyTroupe) simulent les scénarios de risque : "Si la dépression de l'enceinte tombe en-dessous de 5 Pa, voici ce qui se passe." Alertes proactives.

**Pourquoi Robin:** Expertise unique en diagnostic polluants. Mais IoT + capteurs = effort hardware important.

**Concurrence:** OpenSpace, Buildots font du suivi chantier visuel. Aucun ne fait du suivi spécifique décontamination avec simulation de risques.

**Premier pas:** POC avec photos + checklist digitale, sans IoT. La simulation vient après.
